# ¿Qué es Markdown?

Markdown es una forma sencilla de escribir y dar formato a textos. Usando símbolos como #, *, -, puedes crear títulos, listas, tablas y más, sin complicarte. Es muy usado en proyectos de software porque permite documentar de manera rápida y clara, sin depender de programas especiales.

# ¿Por qué se usa en proyectos de software?

Porque es fácil de leer, de escribir y de mantener. Los archivos Markdown (como los README.md) ayudan a que todos entiendan cómo funciona un proyecto, cómo instalarlo o cómo contribuir.

# Ejemplo práctico de Markdown

## Títulos
# Título grande
## Título mediano
### Título pequeño

## Listas
- Elemento 1
- Elemento 2
  - Sub-elemento

1. Paso uno
2. Paso dos

## Tabla
| Nombre | Edad |
|--------|------|
| Ana    | 23   |
| Juan   | 30   |

## Enlaces
[Google](https://www.google.com)

# Ventajas de usar Markdown con GitHub

- Es fácil de aprender y usar.
- Puedes ver la documentación directamente en GitHub, sin descargar nada.
- Varias personas pueden editar los archivos y ver los cambios.
- Los archivos pesan poco y funcionan en cualquier sistema.
- Se integra con muchas herramientas de desarrollo.
